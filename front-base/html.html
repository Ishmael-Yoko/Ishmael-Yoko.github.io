      HTML(HeyperText Markup Language) 是描述网页结构的超文本标记语言，HTML规范能够使HTML代码的风格保持一致，使得HTML更容易理解和维护。

      # 1. HTML 规范

      # 整体结构

      1. 文件应该以<!DOCTYPE ...>的首行顶格开始，推荐使用<!DOCTYPE html>
       <!DOCTYPE html>
      1. 必须声明文档的charset，并且与文本本身编码保持一致，制定支付编码的meta必须是head的第一个直接子元素，推荐使用UTF-8 编码
       <!DOCTYPE html>
 <html lang='en'>
   <head>
     <meta charset='utf-8'>
   </head>
 </html>
      1. 根据页面内容和需求填写适当的keywords和description
       <!DOCTYPE html>
 <html lang='en'>
   <head>
     <meta charset='utf-8'>
     <meta name='description' content='不超过150个字符'>
     <meta name='keywords' content=''>
   </head>
 </html>
      1. 页面title是不可缺少的一项，title必须作为head的直接子元素，并在charset之后。title可以通过js的window.document.title= '标题' 设置
        <!DOCTYPE html>
  <html lang='en'>
    <head>
      <meta charset='utf-8'>
      <title>页面标题</title>
      <meta name='description' content='不超过150个字符'>
      <meta name='keywords' content=''>
    </head>
  </html>
      1. 资源引入
      <!-- favico 引入 -->
<link rel='icon' href='path/to/favicon.ico'>
<!-- css js 引入 -->
<link href='path/to/css.css' rel='stylesheet'>
<script src='path/to/js.js'>

      rel 的取值有多种

      值 描述
      alternate 文档的替代版本（比如打印页、翻译或镜像）
      stylesheet 文档的外部样式表
      contents 文档的目录
      start 集合中的第一个文档
      next 集合中的下一个文档
      prev 集合中的上一个文档
      index 文档的索引
      glossary 在文档中使用的词汇的术语表（解释）
      copyright 包含版权信息的文档
      chapter 文档的章
      section 文档的节
      subsection 文档的小节
      appendix 文档的附录
      help 帮助文档
      preload 预加载在 CSS 和JavaScript 中定义的资源(资源提示 H5)
      prefetch 浏览器在后台（空闲时）获取将来可能用得到的资源，并且将他们存储在浏览器的缓存中(资源提示 H5)
      dns-prefetch 在后台运行 DNS 的解析(资源提示 H5)
      prerender 后台渲染了整个页面(资源提示 H5)
      preconnect HTTP 请求正式发给服务器前预先执行一些操作(资源提示 H5)

      注意

      1. css 和 js 的引入无需指定type
      2. link引入css 置于head之中，使用script引入js置于body底部
      3. 移动环境或者只针对现代浏览器设计的web应用，如果引入外部资源的URL协议部分与页面相同（http/https），建议省略协议前缀。因为这种情况在IE7/8会发两次请求。
      4. 只有 rel 属性的 "stylesheet" 值得到了所有浏览器的支持

      结构优化

      1. 尽量遵循 HTML 标准和语义，但是不要以牺牲实用性为代价。任何时候都要尽量使用最少的标签并保持最小的复杂度
      2. 结构顺序和视觉顺序基本保持一致，按照从上至下、从左到右的视觉顺序书写HTML结构。有时为了便于搜索引擎抓取，也会将重要内容在HTML结构顺序上提前
      3. 结构(html)、表现(css)、行为(js)三者分离，避免内联
      4. 每一个块级元素都另起一行，每一行都使用Tab缩进对齐（head和body的子元素不需要缩进）。删除冗余的行尾空格
      5. 对于内容较为简单的表格，建议将tr写成单行
      6. 可以在大的模块之间用空行隔开，使模块更清晰

      语义化

      浏览器会根据标签的语义给定一个默认的样式。判断网页标签语义化是否良好的一个简单方法：去掉样式，看网页结构是否组织良好有序，是否仍然有很好的可读性

      1. 尽可能少地使用无语义标签span和div
      2. 在语义不明显，既可以使用p也可以使用div的地方，尽量用p
      3. 在既可以使用div也可以使用section的地方，尽量用section
      4. 不要使用纯样式标签，如b、u等，而改用CSS设置

      # 代码格式

      1. 使用2个空格代替1个Tab（大多数编辑器中可设置）
      2. 选择器的命名参考 推荐使用BEM的规范

      BEM

      使用 BEM 命名规范，理论上讲，每行 css 代码都只有一个选择器。
      BEM代表 “块（block）,元素（element）,修饰符（modifier）”,我们常用这三个实体开发组件

      在选择器中，由以下三种符号来表示扩展的关系：

      -   中划线 ：仅作为连字符使用，表示某个块或者某个子元素的多单词之间的连接记号。
__  双下划线：双下划线用来连接块和块的子元素
_   单下划线：单下划线用来描述一个块或者块的子元素的一种状态

type-block__element_modifier

      BEM 解决问题: 组件之间的完全解耦，不会造成命名空间的污染，如：.mod-xxx ul li 的写法带来的潜在的嵌套风险。

      1. 同一页面，应避免使用相同的 name 与 id。因为IE7-浏览器会混淆元素的 id 和 name 属性， document.getElementById 可能获得不期望的元素。所以在对元素的 id 与 name 属性的命名需要非常小心

      2. 注释：采用类似标签闭合的写法，与HTML统一格式；注释文案两头空格，与CSS注释统一格式

      <!-- 头部 -->
<div class="g-hd">
    <!-- LOGO -->
    <h1 class="m-logo"><a href="#">LOGO</a></h1>
    <!-- /LOGO -->
    <!-- 导航 -->
    <ul class="m-nav">
        <li><a href="#">NAV1</a></li>
        <li><a href="#">NAV2</a></li>
        <!-- 更多导航项 -->
    </ul>
    <!-- /导航 -->
</div>
<!-- /头部 -->
      1. 自定义属性建议以xxx- 为前缀，推荐使用 data-xxx 自定义属性的值获取和设置：建议使用dom.getAttribute("data-xxx")和dom.setAttribute("data-xxx", 'aaa')

      # 特殊元素

      1. 禁止img和src取值为空，否则会导致部分浏览器重新加载一次当前页面
      2. 为图片添加alt属性，提高图标加载失败时用户体验
      3. 避免img添加不必要的title属性，多余的title影响看图体验
      4. 为图片添加width和height 避免页面抖动
      5. 有下载需求的图片采用img实现，无下载需求的图片采用css背景实现
      6. 有文本标题的控件使用 label 标签将其与其标题相关联。最好将控件置于 label 内，以减少不必要的 id
      <label>
  <input type="checkbox" name="confirm" value="on"> 我已确认上述条款
</label>
      1. 使用 button 元素时必须指明 type 属性值。因为button 元素的默认 type 为 submit，如果被置于 form 元素中，点击后将导致表单提交
      <button type="submit">提交</button>
<button type="button">取消</button>
      1. 在针对移动设备开发的页面时，根据内容类型指定输入框的 type 属性，能获得友好的输入体验
      <input type="date">
      1. 在支持 HTML5 的浏览器中优先使用 audio 和 video 标签来定义音视频元素，并使用退化到插件的方式来对多浏览器进行支持

      # 2. HTML条件注释

      条件注释是IE支持的一种HTML非标准逻辑语句，作用是灵活的为不同版本的IE浏览器导入不同的HTML元素。从IE5就开始支持。

      从IE10开始，IE浏览器应不再支持条件注释（微软自己估计都看不下去了）

      <!--[if IE]>
<div class="box">这是一个div</div>
<![endif]-->
      • 识别单一IE
      <!--[if IE6]>
<div class="box">这是一个div</div>
<![endif]-->
<!--[if IE7]>
<div class="box">这是一个div</div>
<![endif]-->
<!--[if IE8]>
<div class="box">这是一个div</div>
<![endif]-->
<!--[if IE9]>
<div class="box">这是一个div</div>
<![endif]-->
      • 识别范围IE
      gt        　大于(greater than)
gte     　　 大于等于(greater than or equal)
lt          小于(less than)
lte         小于等于(less than or equal)

      <!--[if lte IE7]>
<div class="box">这是一个div</div>
<![endif]-->
<!--[if gte IE8]>
<div class="box">这是一个div</div>
<![endif]-->
      • 非IE识别

      识别的是IE10+ 和其他非IE的浏览器

      <!--[if !IE]>
<div class="box">这是一个div</div>
<![endif]-->

      # 3. HTML文档的头部

      HTML的头部指的就是head标签中内容。head大部分不可见，描述了文档的一些基本的属性和信息（可以展现的就是title和icon）。下面主要包括meta、title、base、link、style、script六个元素

      # meta

      meta标签用于提供页面有关的数据，除了提供文档字符集、使用语言、作者等基本信息，还设计对关键词和网页等级的设定。通过设置不同属性，元数据可以分为以下几种

      • 如果设置了charset，将对网页使用的字符集做出声明
      • 如果设置了name，它是一个文档级的元数据，将附着在真个页面上
      • 如果设置了http-equiv，它是一个编译指令，将由服务器提供的来指示页面如何加载
      • 如果设置itemprop，将定义一个用户自定义的元数据（未实现）
      1. charset

      charset声明当前文档所使用的的字符编码，但该声明可以被任何一个元素的lang特性值覆盖。文档的编码一定要与文件本身的编码保持一致，否则会出现乱码，一般情况推荐使用UTF-8

      <meta charset='utf-8'/>
      1. name

      [关键字]

      <meta name='keywords' content='html,css,xml'/>

      [描述]

      <meta name='description' content='this is a description'/>

      [作者]

      <meta name='author' content='Ishmael'/>

      [版权]

      <meta name='copyright' content='本页版权归Ishmael所有'>

      [视口]

      <meta name='viewport' content='width=device-width,inital-scale=1.0,maximum-scale=1.0'>

      [双核浏览器渲染]

      <meta name='renderer' content='webkit'>
<!-- 如果是双核浏览器使用webkit内核渲染 -->
      1. http-equiv

      [IE 浏览器渲染]

      <meta http-equiv='X-UA-Compatible' content="IE=edge,chrome=1">

      上面这段代码的意思是，如果安装了GCF（Google Chrome Fram谷歌内嵌浏览器框架GCF），则使用GCF来渲染页面，如果没有安装，则使用最高版本的IE内核渲染页面。

      [定时跳转]

      <meta http-equiv='refresh' content='3'>
<meta http-equiv='refresh' content='3,url=http://www.baidu.com'>

      [禁止缓存]

      <!-- http/1.0 -->
<meta http-equiv='Pragma' content='no-cache'>
<!-- http/1.1 -->
<meta http-equiv='Cache-Control' content='no-cache'>

      [缓存过期时间]

      <!-- http/1.0 -->
<meta http-equiv='Expires' content='Sat Nov 28 2016 21:19:15 GMT+0800'>
<!-- http/1.1 -->
<meta http-equiv='Cache-Control' content='Sat Nov 28 2016 21:19:15 GMT+0800'>

      [独立页面]

      <meta http-equiv='Window-Target' content='_top'>
<!-- 强制页面以独立页面展示，防止自己网页被别人当作iframe调用 -->

      关于meta的更多信息请看meta参考

      # title

      title 元素定义文档的标题，显示在浏览器的标题栏或标签页上。它只可以包含文本，若是包含有标签，则包含的任何标签都不会被解释。

      title 元素是页面必须的，有三个作用：

      1. 浏览器工具栏的标题
      2. 页面到收藏夹的标题
      3. 显示在搜索引擎中的页面标题

      # base

      base 元素用于指定文档里所有相对URL地址的基础URL，为页面上所有链接规定默认地址和默认打开方式。文档中最多只可以出现一个base标签，如果有多个，只会使用第一个指定的href和target方式

      <base href='http://ishmael-yoko.github.io' target='_blank'>

      # link

      link元素规定了当前文档与外部资源的关系。该元素最常用于链接样式表，此外也可以被用来创建站点图标(比如PC端的“favicon”图标和移动设备上用以显示在主屏幕的图标) 。

      <link href="/media/examples/link-element-example.css" rel="stylesheet">

      • rel rel属性指明被链接文档对于当前文档的关系。这个属性是由一个空格分开的链接类型值的列表。 支持的值有

      • href 此属性指定被链接资源的URL。 URL 可以是绝对的，也可以是相对的

      • as 该属性仅在link元素设置了 rel="preload" 时才能使用。它规定了元素加载的内容的类型，对于内容的优先级、请求匹配、正确的内容安全策略的选择以及正确的 Accept请求头的设置，这个属性是必需的

      • crossorigin 此枚举属性指定在加载相关图片时是否必须使用 CORS 取值:

        • anonymous 会发起一个跨域请求。但不会发送任何认证信息，如果服务器没有设置Access_control-Allow-Origin:HTTP头，这个图片就会被污染
        • use-credentials：会发起一个带有认证信息（cookie）的跨域请求。同样服务器没有设置origin头，图片就hi被污染

      link的更多用法，请参考link参考

      # style

      加载外部的样式表

      # script

      加载外部的可执行脚本

      [src]

      这个属性定义引用外部脚本的URI，这可以用来代替直接在文档中嵌入脚本。有src属性的script元素标签内不应该再有嵌入的脚本

      [type]

      该属性定义script元素包含或src引用的脚本语言。属性的值为MIME类型，支持的MIME类型包括text/javascript, text/ecmascript, application/javascript和application/ecmascript。

      [defer]

      在解析DOM的时候异步的加载外部js文件，当DOM解析完全后，执行对应js脚本

      [async]

      在解析DOM的时候异步加载外部js文件，js加载完成后立即执行js脚本，此时无论DOM的解析是否完成都会被暂停，直到js执行完成之后，再继续进行DOM解析

      上面两者的区别：执行时的不同：async脚本在script文件下载完成后会立即执行，并且其执行时间一定在window的load事件触发之前。这意味着多个async脚本很可能不会按其在页面中的出现次序顺序执行；与此相对，浏览器确保多个defer脚本按其在HTML页面中的出现顺序依次执行，且执行时机为DOM解析完成后，document的DOMContentLoaded事件触发之前

      [注意] 如果同时设置async和defer，和只设置async属性的效果一致

      # 语义化

      • HTML5新增的外观语义化标签section、article、nav、header、footer、aside、hgroup

        • 为什么要语义化？
          1. 为了没有css的情况下，页面也能呈现很好的内容结构
          2. 用户体验：title alt 等解释说明，便于用户理解
          3. 有利用SEO: 和搜索引擎建立良好沟通，利于爬虫
          4. 方便其他设备解析：如屏幕阅读器，移动设备等
          5. 便于团队的开发和维护，语义化更具有可读性，减少差异化

      • HTML5的多媒体支持：audio、video

        • <audio>标签

          <!-- autoplay   自动播放
controls   显示控件
loop       循环播放
preload    音频在页面加载时进行加载，并预备播放（若使用autoplay则忽略此属性）
src        要播放的音频的URL -->

<audio controls autoplay loop src='song.mp3'>
  <source src='song.mp3' type='audio/mp3' />
</audio>

          注意

          <audio>元素不支持播放wma格式的文件

        • <video>标签

          <!-- autoplay   自动播放
controls   显示控件
loop       循环播放

preload='auto' 页面加载时载入整个视频
preload='meta' 页面加载时只载入元数据
src        要播放的频视频的URL 
height     播放高度
width      播放宽度
poster     视频下载或者播放前的预览图像
-->

  <source src='movie.mp4' type='video/mp4' />
</video>

          <video>
  <source src="video.webm" type="video/webm; codecs='vp8,vorbis'">
  <source src="video.ogg" type="video/ogg; codecs='theora,vorbis'">
  <source src="video.mp4">
  video player not available.

      • requestAnimationFrame: 允许控制动画渲染以获得更优性能

        • requestAnimationFrame是什么？
          告诉浏览器---你希望执行一个动画，并且要求这个动画在浏览器下次重新绘制之前调用指定的回调函数更新动画。这个函数需要传入一个回调函数作为参数，改回调函数在浏览器下次重新绘制之前执行。

          注意

          如果你想在浏览器下次重新绘制之前继续更新下一帧动画，那么回调函数自身必须再次调用requestAnimationFrame方法。

        • 为什么要使用requestAnimationFrame？
          随着前端发展，css已经可以完成十分复杂的动画特效，但是更为精细的动画还是需要js来完成。一般的解决方案都是使用setInterval，但是setInterval真的就可以吗？我们来下面代码

        let start = +new Date()
setTimeout(() => {
  let end = +new Date()
  console.log(end - start)
})
for (let i = 0; i < 30000; i++) {
  console.log(1)
}

        结果如下：

        可以看到未设置延时，但是执行结果却在2985ms之后打印。因为js是单线程，使用的事件循环机制，setTimeout为宏任务，会加入宏事件列表中，当前执行栈中的代码继续执行，当执行栈中的代码执行完之后再执行时间列表中的代码。具体的事件循环机制后续js专题会讲到，这里不单独讲解。 由此可见，如果按照上面的代码执行动画，效果可想而知。所以requestAnimationFrame的出现就是为了解决这个问题的。 可以理解为：专门执行动画的api，当js中执行这个函数的时候，就是告诉浏览器，要执行动画了，请在每一帧开始之前执行动画。

        let start = null
var ele = document.querySelector('.app')
ele.style.position = 'absolute'

function step(time) {
  if (!start) {
    start = time
  }
  let pro = time - start
  ele.style.left = Math.min(pro / 10, 200) + 'px'
  if (ele.style.left < 30000>) {
    window.requestAnimationFrame(step)
  }
  window.requestAnimationFrame(step)
}
        • requestAnimationFrame中如果存在大量计算，导致动画未在1000s / 60Hz = 16ms完成，在下一帧会继续执行，还是舍弃后重新执行？
          舍弃掉。如果存在大量计算，导致动画未在1000s / 60Hz = 16ms完成，进入下一帧前，没有执行的就会舍弃掉，重新执行。
        • requestAnimationFrame实现原理？
          requestAnimationFrame最大的优势是由系统来决定回调函数的执行时机。具体一点讲，如果屏幕刷新率是60Hz,那么回调函数就每16.7ms被执行一次，如果刷新率是75Hz，那么这个时间间隔就变成了1000/75=13.3ms，换句话说就是，requestAnimationFrame的步伐跟着系统的刷新步伐走。它能保证回调函数在屏幕每一次的刷新间隔中只被执行一次，这样就不会引起丢帧现象，也不会导致动画出现卡顿的问题。
        • requestAnimationFrame优势？
          当页面处理未激活的状态下，该页面的屏幕刷新任务也会被系统暂停，因此跟着系统步伐走的requestAnimationFrame也会停止渲染，当页面被激活时，动画就从上次停留的地方继续执行，有效节省了CPU开销。

      • 全屏 API: 全屏 API 为使用用户的整个屏幕展现网络内容提供了一种简单的方式。这种API让你可以简单地控制浏览器，使得一个元素与其子元素，如果存在的话，可以占据整个屏幕，并在此期间，从屏幕上隐藏所有的浏览器用户界面以及其他应用。

        全屏API在Document和Element接口中增加了一些方法，使其可以打开关闭全屏模式。

        • Document.exitFullscreen(): 此方法使其退出全屏模式，这个方法会使文档退回到使用Element.requestFullScreen()变成全屏状态之前的状态。返回一个Promise
        • Element.requestFullscreen(): 此方法是异步请求浏览器使其置为全屏模式。返回一个Promise
        • Document.fullscreenElement: 此属性提供了当前DOM被置为全屏模式的Element。如果这个值为null，这个文档不处于全屏模式
        • Document.fullscreenEnabled: 此属性提供了是否支持全屏值。false不支持true为支持。
        • fullscreenchange:在DOM进入全屏，或者退出全屏时触发。此事件可以在document上注册，也可以在需要全屏的element上注册，但是我们一般注册在document上，如果只全屏部分DOM，也可以通过事件冒泡触发
        • fullscreenerror: 当进入或退出全屏时出错，会触发此事件。同样我们通过事件冒泡注册在document上.

          以上是HTML5规范中的API，但是不是所有的浏览器都是使用相同名称的API。不同浏览器存在不同的兼容性。

        // 立即执行函数IIFE包裹，避免全局污染
(function () {
  'use strict';
  // 判断是否是浏览器环境（script 引入），是的话获取文档模型
  var document = typeof window !== 'undefined' && typeof window.document !== 'undefined' ? window.document : {};
  // 判断是否是commonJS环境（npm 引入）
  var isCommonjs = typeof module !== 'undefined' && module.exports;
  // fn 函数 提供各个浏览器全屏的方法、属性、事件。最终获取当前浏览器环境的方法，属性，事件
  var fn = (function () {
    var val;
    //各个浏览器的方法、属性、事件
    var fnMap = [
      [
        'requestFullscreen',
        'exitFullscreen',
        'fullscreenElement',
        'fullscreenEnabled',
        'fullscreenchange',
        'fullscreenerror'
      ],
      // New WebKit
      [
        'webkitRequestFullscreen',
        'webkitExitFullscreen',
        'webkitFullscreenElement',
        'webkitFullscreenEnabled',
        'webkitfullscreenchange',
        'webkitfullscreenerror'

      ],
      // Old WebKit
      [
        'webkitRequestFullScreen',
        'webkitCancelFullScreen',
        'webkitCurrentFullScreenElement',
        'webkitCancelFullScreen',
        'webkitfullscreenchange',
        'webkitfullscreenerror'

      ],
      [
        'mozRequestFullScreen',
        'mozCancelFullScreen',
        'mozFullScreenElement',
        'mozFullScreenEnabled',
        'mozfullscreenchange',
        'mozfullscreenerror'
      ],
      [
        'msRequestFullscreen',
        'msExitFullscreen',
        'msFullscreenElement',
        'msFullscreenEnabled',
        'MSFullscreenChange',
        'MSFullscreenError'
      ]
    ];

    var i = 0;
    var l = fnMap.length;
    var ret = {};
    // 获取当前浏览器的全屏API方法、属性、事件
    for (; i < l; i++) {
      val = fnMap[i];
      if (val && val[1] in document) {
        for (i = 0; i < val.length; i++) {
          ret[fnMap[0][i]] = val[i];
        }
        return ret;
      }
    }
    // 当前浏览器不支持全屏，直接返回false
    return false;
  })();
  // 全屏事件的映射。change事件映射到全屏fullscreenchange事件，error事件映射为全屏的fullscreenerror事件
  var eventNameMap = {
    change: fn.fullscreenchange,
    error: fn.fullscreenerror
  };
  // 封装全屏对象
  var screenfull = {
    // 全屏方法
    request: function (element) {
      // 用promise封装异步
      return new Promise(function (resolve, reject) {
        // 全屏change事件，当进入全屏的时候，去掉全屏change事件的监测。
        var onFullScreenEntered = function () {
          this.off('change', onFullScreenEntered);
          resolve();
        }.bind(this);
        // 注册全屏change事件，进入全屏时执行
        this.on('change', onFullScreenEntered);
        // 要全屏的DOM，传入参数则使用该参数，不传参数则为document
        element = element || document.documentElement;
        // promise封装异步执行进入全屏状态
        Promise.resolve(element[fn.requestFullscreen]()).catch(reject);
        // 使用bind将this指向screenfull对象
      }.bind(this));
    },
    // 退出全屏方法
    exit: function () {
      // 用promise封装异步方法
      return new Promise(function (resolve, reject) {
        // 如果是非全屏状态则什么都不处理
        if (!this.isFullscreen) {
          resolve();
          return;
        }
        // 全屏退出change事件的回调，当退出全屏的时候，去掉change事件的监测
        var onFullScreenExit = function () {
          this.off('change', onFullScreenExit);
          resolve();
        }.bind(this);
        // 注册全屏change事件，退出全屏时执行
        this.on('change', onFullScreenExit);
        // promise封装异步执行退出全屏状态
        Promise.resolve(document[fn.exitFullscreen]()).catch(reject);
        / 使用bind将this指向screenfull对象
      }.bind(this));
    },
    // 切换指定DOM的全屏状态
    toggle: function (element) {
      return this.isFullscreen ? this.exit() : this.request(element);
    },
    // 注册全屏change事件
    onchange: function (callback) {
      this.on('change', callback);
    },
    // 注册全屏切换出错时的事件
    onerror: function (callback) {
      this.on('error', callback);
    },
    // 用于注册全屏事件(fullscreenchange fullscreenerror)的入口
    on: function (event, callback) {
      // 用于映射到内部事件map,防止注册非法事件
      var eventName = eventNameMap[event];
      if (eventName) {
        // 注册事件。所有事件注册到document 是通过事件冒泡的方式，任何dom的全屏状态均可响应（使用了事件代理）
        document.addEventListener(eventName, callback, false);
      }
    },
    // 去掉注册事件
    off: function (event, callback) {
      // 用于映射到内部事件map,防止注册非法事件
      var eventName = eventNameMap[event];
      if (eventName) {
        // 注销事件。因为都在document上注册，直接在document上注销即可
        document.removeEventListener(eventName, callback, false);
      }
    },
    // 返回原生的浏览器的全屏的方法，事件，属性 对象。用于自定义使用
    raw: fn
  };
  // 浏览器不支持全屏，则直接返回isEnable:false
  if (!fn) {
    // commonJS 环境 直接返回对象
    if (isCommonjs) {
      module.exports = {isEnabled: false};
    } else {
      // 浏览器环境则将对象挂载至window
      window.screenfull = {isEnabled: false};
    }

    return;
  }
  // screenfull对象的劫持处理。增加新的属性
  Object.defineProperties(screenfull, {
    //增加isFullscreen属性，是否是全屏状态，document[fn.fullscreenElement]为null则不存在
    isFullscreen: {
      get: function () {
        return Boolean(document[fn.fullscreenElement]);
      }
    },
    // 返回全屏的DOM对象
    element: {
      enumerable: true,
      get: function () {
        return document[fn.fullscreenElement];
      }
    },
    // 返回是否浏览器是否支持全屏
    isEnabled: {
      enumerable: true,
      get: function () {
        // Coerce to boolean in case of old WebKit
        return Boolean(document[fn.fullscreenEnabled]);
      }
    }
  });
  // commonJS 环境 直接返回screenfull对象
  if (isCommonjs) {
    module.exports = screenfull;
  } else {
    // 浏览器环境则将screenfull对象挂载至window
    window.screenfull = screenfull;
  }
})();

  var accelerationIncludingGravity = DeviceMotionEvent.accelerationIncludingGravity;

  // beta: 设备沿着屏幕左至右方向的轴的旋转速率(桌面设备相对于键盘)
  // gamma: 设备沿着屏幕下至上方向的轴的旋转速率(桌面设备相对于键盘)
  var rotationRate = DeviceMotionEvent.rotationRate;

}
